# Roadmap

## Week 1: Canonical core

- Ship canonical `v1/analyze`, `v1/emergency`, and `v1/rails`.
- Lock one response contract and publish JSON schemas.
- Replace the Expo toy analyzer with backend-first flows.
- Add red-team tests for fake KYC APK, QR refund trap, digital arrest, and thin-evidence handling.

## Week 2: Capture and intervene

- Add share-to-Chetana on Android.
- Add QR decode before open or pay.
- Add screenshot OCR entry path.
- Add family warning card and merchant QR shield.

## Week 3: Device risk and trust layers

- Add APK metadata extraction into the app flow.
- Add suspicious APK hash capture and evidence bundle export.
- Add multilingual copy: English, Hindi, Tamil first.
- Add attribution and source tagging for the distribution loop.

## Week 4: Controlled beta

- Notification listener opt-in.
- Closed beta on OnePlus + Pixel devices.
- Operator dashboard for top threat lanes and emergency triggers.
- Guardian / family review mode.

## Phase 2

- Merchant mode with payment-proof dispute flow.
- Elder mode with simplified UI and guardian escalation.
- Business mode for vendor-bank-change and callback verification.
- Scam-pattern feed with signed updates.

## Phase 3

- On-device lightweight classifier and offline fallback.
- Enterprise or device-owner mode where policy allows.
- Bank, PSP, lender, and police escalation partnerships if they become available.
